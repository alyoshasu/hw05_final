# «Yatube»
Социальная сеть для публикации личных дневников.<br>

Это сайт, на котором можно создать свою страницу. Если на нее зайти, то можно посмотреть все записи автора.<br>

Пользователи могут заходить на чужие страницы, подписываться на авторов и комментировать их записи.<br>

Автор может выбрать имя и уникальный адрес для своей страницы.<br>

Имеется возможность модерировать записи и блокировать пользователей, если начнут присылать спам.<br>

Пользователь может создать, отредактировать или удалить запись.<br>

Записи можно опубликовать не только у себя в ленте, но и выбрать группу (сообщество), в которой появится этот пост.<br>

Пользователь может зайти в группу (сообщество) и посмотреть там записи разных авторов.<br>

Группы (сообщества) создаются администрацией сайта, посетители не смогут их добавлять.<br>

## Покрытие тестами

### Тестирование Models
Тест, проверяющий содержание verbose и help_text в полях моделей Post и Group (используются в форме PostForm);<br>

Тест, проверяющий, правильно ли отображается значение поля __str__ в объектах моделей:
- для класса Post — первые пятнадцать символов поста: post.text[:15],
- для класса Group — название группы.<br>

### Тестирование URLs
Доступность страниц в соответствии с правами пользователей.<br>
- главная страница ```/```,
- страница группы ```/group/<slug>/```,
- страница создания поста ```/new/```,
- профайла пользователя ```/<username>/```,
- отдельного поста ```/<username>/<post_id>/```;<br>

Доступность страницы редактирования поста ```/<username>/<post_id>/edit/``` для:
- анонимного пользователя,
- авторизованного пользователя — автора поста,
- авторизованного пользователя — не автора поста;<br>

Тест, проверяющий, какой шаблон вызывается для страницы редактирования поста ```/<username>/<post_id>/edit/```;<br>

Тест, проверяющий, правильно ли работает редирект со страницы ```/<username>/<post_id>/edit/``` для тех, у кого нет прав доступа к этой странице;<br>

Обращение по URL к следующим страницам и проверка, вызываются ли для них ожидаемые шаблоны:
- главная страница ```/```,
- страница группы ```/group/<slug>/```,
- страница создания поста ```/new/```.<br>

### Тестирование Views
Тесты, проверяющие, какой шаблон будет вызван при обращении к view-классам через соответствующий name для:
- главной страницы,
- страницы группы,
- страницы создания поста;<br>

Тесты, проверяющие, соответствует ли ожиданиям словарь context, передаваемый в шаблон при вызове:
- главной страницы,
- страницы группы,
- страница создания поста;<br>

Тесты, проверяющие, что если при создании поста указать группу, то этот пост появляется
- на главной странице сайта,
- на странице выбранной группы;<br>

Проверка, что этот пост не попал в группу, для которой не был предназначен;<br>

Тесты содержимого словаря context для страниц:
- редактирования поста ```/<username>/<post_id>/edit/```,
- профайла пользователя ```/<username>/```,
- отдельного поста ```/<username>/<post_id>/```;<br>

Тест паджинатор: 
- проверка, что в словарь context главной страницы передаётся не более установленного количества постов.<br>

### Тестирование Forms

Тест формы создания нового поста (страница ```/new/```):
- при отправке формы создаётся новая запись в базе данных.<br>

Проверка, что при редактировании поста через форму на странице ```/<username>/<post_id>/edit/``` изменяется соответствующая запись в базе данных.
